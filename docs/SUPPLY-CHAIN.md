# Supply Chain Trust Model

**Owner:** `@greenmetrics/secops`.
**Doctrine refs:** Rule 53 (supply chain is core asset), Rule 54 (policy as code), Rule 56 (automation), Rule 65 (regulated quality), Rule 67 (transparency).
**Plan ADRs:** `docs/adr/0017-cosign-keyless-oidc.md`, `docs/adr/0018-slsa-l2-now-l3-plan.md`.

This document is the operator's reference for how GreenMetrics artefacts get from source to running cluster, what is verified at each step, and what residual risks remain.

## 1. Trust chain

```
operator commit
   │
   │  git push (signed via GitHub web auth or SSH key)
   ▼
GitHub repo
   │
   │  PR review (CODEOWNERS) + branch protection (required checks)
   ▼
GitHub Actions runner (ephemeral)
   │
   │  GitHub OIDC token (id-token: write permission)
   ▼
Sigstore Fulcio (issues short-lived signing cert bound to workflow identity)
   │
   │  cosign sign --yes ghcr.io/<repo>/<image>:<sha>
   ▼
Sigstore Rekor (transparency log entry)
   │
   │  cosign attest --type spdx (SBOM) + slsa-github-generator (provenance)
   ▼
GHCR (image + signature + attestations)
   │
   │  Argo CD pull (no manual kubectl)
   ▼
EKS kubelet
   │
   │  Kyverno admission verify-images against Sigstore Fulcio cert + workflow identity
   ▼
Pod runs
```

## 2. SLSA target

- **Today (S3 close):** SLSA L2.
  - Provenance generated by `slsa-github-generator` (hosted GitHub-controlled builder).
  - Signed via Cosign keyless (Sigstore Fulcio + Rekor transparency).
  - SBOM attached as Cosign attestation (SPDX format from Syft).
  - Kyverno admission policy `policies/kyverno/verify-images.yaml` denies pods whose image lacks valid Sigstore signature bound to the GitHub workflow `subject` filter.
- **Plan (next half):** SLSA L3.
  - Hermetic, fully reproducible builds.
  - Two-party review on every workflow file change.
  - External audit of provenance chain.

ADR: `docs/adr/0018-slsa-l2-now-l3-plan.md`.

## 3. Per-stage verification

| Stage | What is verified | Mechanism |
|---|---|---|
| Source | Authorial intent, code review | CODEOWNERS routing, signed commits, branch protection |
| Build | Reproducible build inputs | GitHub-hosted runner, pinned Action SHAs (Dependabot-managed), Go modules pinned in `go.sum`, npm pinned in `package-lock.json`, Docker base images digest-pinned |
| SAST | Code-level vulnerabilities | CodeQL (security-extended), Semgrep (p/default + p/owasp-top-ten + p/golang), `govulncheck`, `osv-scanner` (cross-source), `gosec` (in golangci-lint), gitleaks (secrets) |
| Dependency | Known CVEs | govulncheck + osv-scanner + Trivy FS + Trivy Image + Dependabot weekly bumps + license check (`go-licenses`, npm `license-checker`) against `LICENSES.allowed` |
| Container | Image-level vulnerabilities | Trivy image scan post-push (HIGH/CRITICAL block) |
| Artefact | Supply integrity | Cosign keyless sign (Sigstore Fulcio cert bound to workflow identity), Cosign attest SBOM (SPDX), SLSA L2 provenance attest |
| Deploy | Image authenticity | Argo CD pulls from GHCR by digest; Kyverno admission `verify-images` checks Cosign signature against expected workflow identity before pod creation |
| Run | Runtime behaviour | Falco / Tetragon DaemonSet flags out-of-policy process, network, file activity; events ship to Loki + Alertmanager |

## 4. GitHub Actions pinning policy

Every `uses:` in `.github/workflows/*.yml` should reference a commit SHA, not a floating tag. Format:

```yaml
- uses: actions/checkout@<40-char-sha>  # v4.X.Y
```

Today (S1 ship): workflows reference floating tags for readability. Dependabot is configured to migrate every reference to a pinned SHA on its next weekly run (see `.github/dependabot.yml` ecosystem `github-actions`). Until that PR lands, supply-chain risk is documented as `RISK-005` in `docs/RISK-REGISTER.md` with a 14-day mitigation window.

After Dependabot migration, the convention is enforced:

- New `uses:` in PRs without SHA + comment fail the `actions-lint` CI job (custom rule via `actionlint` config).
- Quarterly secops review verifies pins still resolve.

## 5. Container base image policy

- Backend `backend/Dockerfile` already digest-pins distroless base.
- Frontend `frontend/Dockerfile` ships with floating `node:20-alpine` tag — closing in S1 batch via Renovate or manual digest pin (tracked under `RISK-002` for the rolling-apk concern; orthogonal digest pin for the `FROM` reference is closing now).

## 6. Anti-patterns rejected (Rule 66)

- Cosign with custom KMS keys — rejected. Keyless OIDC only. Custom keys add key-management burden without security improvement when GitHub OIDC + Sigstore Fulcio + Rekor transparency are sufficient.
- Vendor-managed image signing service (e.g. Aqua, Sysdig) — rejected. Sigstore is the open standard; tool sprawl is rejected per Rule 66.
- Signing on a developer laptop with a personal key — rejected. Loss of provenance to the build pipeline.
- Skip Trivy image scan to "speed up CD" — rejected. Trivy image scan is the post-build gate; failure blocks promotion.
- Pin to `@latest` for convenience — rejected. SHA pinning is the rule.

## 7. Operator commands

```bash
# Verify a deployed image's signature against Sigstore.
cosign verify ghcr.io/<owner>/greenmetrics-backend@sha256:<digest> \
  --certificate-identity-regexp "^https://github.com/<owner>/greenmetrics" \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com

# Verify SLSA provenance attestation.
cosign verify-attestation \
  --type slsaprovenance \
  --certificate-identity-regexp "^https://github.com/<owner>/greenmetrics" \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  ghcr.io/<owner>/greenmetrics-backend@sha256:<digest>

# Verify SBOM attestation.
cosign verify-attestation \
  --type spdx \
  --certificate-identity-regexp "^https://github.com/<owner>/greenmetrics" \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  ghcr.io/<owner>/greenmetrics-backend@sha256:<digest>

# Test admission denial of an unsigned image (should fail).
kubectl apply -f - <<EOF
apiVersion: v1
kind: Pod
metadata: { name: unsigned-test, namespace: greenmetrics }
spec: { containers: [{ name: c, image: nginx:latest }] }
EOF
# Expect: admission webhook "validate.kyverno.svc" denied the request.
```

## 9. Verification cadence

- Per PR: Trivy FS, govulncheck, osv-scanner, gitleaks, semgrep, CodeQL, license check.
- Per CD: Cosign sign + attest, SLSA provenance generate + attach, Trivy image scan, Kyverno admission verify.
- Weekly: Dependabot ecosystem PRs (gomod, npm, github-actions, docker, terraform).
- Quarterly: secops review of Cosign root, Renovate config, license allowlist, supply-chain health.
- Annually: Sigstore root key trust review.
