## 1. Regulatory scope

- DL 19/2024 conv. L. 56/2024 (Piano Transizione 5.0).
- DM MIMIT–MASE 24 luglio 2024 (attuativo).
- Eligibility: SMEs investing in 4.0/5.0 assets that produce ≥ 3% per process or ≥ 5% per site energy savings.
- Tax credit: 5% / 15% / 20% / 35% / 40% / 45% banded by savings + investment tier.
- Attestazione (sworn statement) by qualified energy expert.

## 3. Determinism guarantee

`internal/domain/reporting/piano_5_0.go` is deterministic:

- Same `(tenant_id, period_from, period_to, baseline_period, factor_versions)` → same output.
- No `time.Now()`, no `math/rand`; uses `clock.Clock` interface mocked in tests.
- Golden-file tests `internal/domain/reporting/testdata/piano_5_0.golden.{html,json}` (S5).

## 4. Operator query

```sql
-- All Piano 5.0 reports generated for an SME.
SELECT r.id, r.period_from, r.period_to, r.status, r.file_url, r.data_freshness,
       a.created_at AS generated_at, a.user_email AS generated_by
FROM reports r
JOIN audit_log a ON a.target = r.id::text
WHERE r.tenant_id = '<tenant>' AND r.type = 'piano_5_0'
ORDER BY r.created_at DESC;
```

## 5. Evidence delivery

For a Piano 5.0 attestazione audit:

- Report PDFs (immutable in S3 reports bucket).
- `audit_log` rows for each generation.
- Emission factor snapshot used (`emission_factors` filtered by `valid_from`/`valid_to`).
- Reading-period coverage proof (`readings` count + completeness ratio).
- Cosign verification of deployed image that generated the report.
- Repo commit SHA at time of generation.

## 6. Anti-patterns rejected

- Float-for-money in any calculation — REJ-13; static check.
- Non-deterministic generation (random ordering, current time injection) — REJ-19; clock injected.
- Cached factors silently used without `data_freshness` stamp — REJ; report explicitly stamped.
