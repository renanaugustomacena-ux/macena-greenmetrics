# Accessibility — GreenMetrics

The GreenMetrics console (SvelteKit) targets **WCAG 2.2 AA** conformance and
is tested with `axe-core` in CI.

## 1. Hard commitments

- Semantic HTML in all routes (`+page.svelte`, `+layout.svelte`).
- Color contrast ≥ 4.5:1 for text; 3:1 for non-text.
- All interactive elements keyboard-reachable with a visible focus ring.
- No information conveyed by color alone.
- All form inputs have explicit `<label>` associations.
- Chart components (energy, carbon, scope breakdown) expose tabular
  fallback via a `Show data` button.
- Language attribute `lang="it"` on root HTML for the Italian console;
  `lang="en"` on the docs.

## 2. CI gates

- `axe-core` scan of the rendered SvelteKit build: 0 serious / 0 critical
  findings. Applied via Playwright `@axe-core/playwright` in CI.

## 3. Manual review checklist

Before any release:

- [ ] Tab-order on the main dashboard reaches every action in ≤ 2 chords.
- [ ] Screen reader announces alerts with a clear severity prefix.
- [ ] Color-blind simulation of the energy vs CO2e dashboards does not
      merge distinct series.
- [ ] Landing page `landing-page/index.html` passes axe-core with 0
      serious / critical.
