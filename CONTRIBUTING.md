## 3. How contributions are accepted

1. Open a GitHub issue describing the proposed change. Wait for the
   Author's written confirmation that the change is in scope.
2. Open a pull request implementing the change.
   - The pull request must comply with all CI gates (lint, test,
     security, policy gates) per [`docs/CONTRIBUTING.md`](./docs/CONTRIBUTING.md).
   - The pull request body must include the following statement signed
     with your full legal name and date:

     > I, [FULL LEGAL NAME], confirm that I have read and accepted the
     > Contributor Terms set out in CONTRIBUTING.md, and I submit this
     > Contribution under those terms. Date: [YYYY-MM-DD].
3. Code review by `@renanaugustomacena-ux` and the routed CODEOWNERS team
   per [`.github/CODEOWNERS`](./.github/CODEOWNERS).
4. The Author may request changes, accept the Contribution, or decline
   the Contribution at the Author's sole discretion.
