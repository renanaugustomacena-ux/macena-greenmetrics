- **Sprint S6 — TEE / Certificati Bianchi Report Pack.** Per Plan §5.4 — the Italian energy-efficiency tradeable-certificate scheme administered by GSE (DM 11/01/2017; updated 2025-2030 by DM MASE 21/07/2025). New `packs/report/tee/` (manifest + CHARTER) and `backend/packs/report/tee/` (Go) deliver the per-project tep-saving + K-factor certificate-issuance calculation. **Algorithm**: ex-ante / ex-post tep delta from FactorBundle scenario inputs (`tee_ex_ante_tep`, `tee_ex_post_tep`); vita utile half determined from `tee_vita_utile_years` + `tee_current_year_in_project` (boundary at exactly half is inclusive on the first-half side per DM 11/01/2017 art. 8 c. 4); K-factor application — **K1 = 1.2** for first half of vita utile, **K2 = 0.8** for second half (vecchio "tau" sostituito da DM 11/01/2017); TEE issued = annual_saving_tep × K_factor (one TEE = 1 tep certified annual saving). **Method selector**: `consuntivo` (default), `standardizzato`, `pppm` via `tee_method` integer factor — output Body field. **Regime version** selector: `dm-2017` default + `dm-mase-2025-2030` selector value 1 — the 2025-2030 selector falls back to DM-2017 K-factor semantics in v1.0.0 with an explicit note that engagement Phase 0 Discovery must verify against the decreto vigente at RVC submission (Pack v1.x.0 minor update to follow regulatory consolidation per Rule 138). **Ineligibility**: non-positive saving → TEE = 0 + ineligibility note (DM 11/01/2017 art. 4). Narrative content (project description, EGE certifier ID, RVC submission ID, vita utile methodology, baseline calculation methodology) is null in the Pack — the engagement-fork's reporting orchestrator injects client-supplied content per Rule 144. EGE / ESCo (UNI CEI 11339 / 11352) or ISO 50001 + responsabile certificato required per Rule 137; engagement Phase 0 Discovery verifies. Pure-function Builder per Rule 91; deterministic JSON per Rule 141; tests anchor Rule 89 (bit-perfect reproducibility), annual saving + saving% math, K1 first-half application, K2 second-half application, exactly-half boundary inclusivity, ineligibility on non-positive saving, all three method selectors, default + DM-MASE-2025-2030 regime selector behaviour, missing-input graceful degradation, narrative null-placeholder shape, and `ctx.Err()` cancellation. - **Sprint S6 — Audit Energetico D.Lgs. 102/2014 Report Pack.** Per Plan §5.4 — the highest commercial-volume Italian regulatory dossier (~10 000 grandi imprese + imprese energivore on a 4-yearly cadence; next mass deadline 2027-12-05). New `packs/report/audit_dlgs102/` (manifest + CHARTER) and `backend/packs/report/audit_dlgs102/` (Go) deliver the quantitative half of the legally-mandated `diagnosi energetica`. Pure-function Builder per Rule 91 producing: (1) **per-vector + per-site energy baseline** (electricity, natural_gas, diesel, petrol, lpg, coal, heavy_fuel_oil), aggregating readings by `MeterID = site_id` and converting primary-unit totals to tep using statutory ENEA / ARERA factors per Rule 132 — every `// Source:` comment block cites D.M. 20/07/2004 + ARERA Delibera 03/08 EEN; (2) **obligation block** (audit type via `audit102_obligation_type` selector — `grande_impresa` / `energivora` / `voluntary`; 4-yearly cadence; `next_deadline_iso`; ISO 50001 / EMAS / ISO 14001 exemption basis); (3) **below-50-tep flag** — both auto-derived from total tep AND assertable via `audit102_below_50_exempt` factor (D.Lgs. 73/2020 → art. 8 c. 3-bis); (4) **EnPI candidates** — total energy intensity = total_kWh / total_floor_m² when `audit102_total_floor_m2` is supplied. Liquid-fuel readings (`l_diesel`, `l_diesel_vehicle`, `l_petrol`) are converted l → kg via default densities (0.835 / 0.745 kg/l per D.M. 20/07/2004) overrideable via `audit102_density_<vector>`. tep-conversion factors are also overrideable via `audit102_tep_factor_<vector>` so engagement Phase 0 Discovery can pin to the audit102.enea.it portal version current at submission. Site-classification (industrial vs tertiary) and the Allegato 2 cluster-selection rules are explicitly left to the orchestrator (engagement-side knowledge of ATECO codes); the Pack surfaces this in `notes`. Improvement-measure list, monitoring-plan KPIs, and the UNI CEI EN 16247-1..4 compliance statement / scope description / site-visit summaries / EGE certifier ID are null in the Pack — the engagement-fork's reporting orchestrator injects client-supplied content per Rule 144. Per Rule 137, EGE / ESCo (UNI CEI 11339 / 11352) or UNI CEI EN 16247-5 auditor counter-signature is verified at engagement Phase 0 Discovery and gates signed-state transition. ENEA portal submission (audit102.enea.it) is a separate engagement-side Pack landing in Phase H Sprint S16. Tests anchor Rule 89 (bit-perfect reproducibility), per-vector + per-site totals, lex-sorted site_id ordering, canonical vector ordering regardless of input order, obligation type / exemption / deadline mapping, below-50-tep flag, EnPI candidate emission, tep-factor + density overrides, l → kg conversion, narrative null-placeholder shape, unclassified-row counting, and `ctx.Err()` cancellation. Capability `report-builder-audit_dlgs102` declared in `config/required-packs.yaml`. Doctrine refs: 89, 90, 91, 95, 97, 131, 132, 137, 141, 144.
- **Sprint S6 — Piano Transizione 5.0 attestation Report Pack.** Per Plan §5.4 — the regulatory centrepiece for Italian industrial-transformation engagements (the €6.3 billion DL 19/2024 art. 38 incentive). New `packs/report/piano_5_0/` (manifest + CHARTER) and `backend/packs/report/piano_5_0/` (Go) deliver the dual-criterion energy-saving classification and the per-bracket tax-credit calculation. **Energy savings**: Pack computes saving percentages from `(baseline − counterfactual) / baseline` for both the *struttura produttiva* criterion (T1 ≥3% / T2 >6% / T3 >10%) and the *processi interessati* criterion (T1 ≥5% / T2 >10% / T3 >15%) per DM 24/07/2024 art. 9 c. 2-3, taking the **higher tier** as the eligible attestation tier and reporting the basis. **Tax credit**: per-bracket × per-tier rate matrix encoded for two regime versions — `lb-2025` (default; L. 207/2024 art. 1 c. 427-429 Legge di Bilancio 2025; two brackets ≤€10M @ 35/40/45% and €10M-€50M @ 5/10/15%) and `dm-2024-07-24` (original three-bracket DM regime, retained for retroactive amendments / historical attestations). Annual €50M cap per beneficiary applied with excess surfaced (DL 19/2024 art. 38 c. 4). **Scenario inputs** flow through the FactorBundle as `piano5_*` keys (baseline_energy_kwh, counterfactual_energy_kwh, baseline_process_kwh, counterfactual_process_kwh, investment_total_eur, regime_version, period_year_eur_cap), reusing the existing FactorBundle channel rather than extending the Builder contract — same convention as ESRS E1's use of `it_renewable_share`. Narrative content (ex-ante / ex-post EGE certifications, Allegato A categorisation, investment description, ATECO codes) is null in the Pack — the engagement-fork's reporting orchestrator injects client-supplied content before signing per Rule 144. Pure-function Builder per Rule 91 — no `time.Now()`, no env, no `internal/services` imports. Every rate constant carries a `// Source:` comment block referencing the primary regulatory publication (Rule 132). Tests anchor Rule 89 (bit-perfect reproducibility), tier-classification thresholds for both criteria, dual-criterion max-of-both logic, LB-2025 + DM-24-07-24 regime rate matrices, annual-cap excess accounting, ineligibility below thresholds, missing-input graceful degradation, narrative-block null-placeholder shape, and `ctx.Err()` cancellation. ## How to add an entry

- Add to `[Unreleased]` first.
- On release: rename `[Unreleased]` to `[X.Y.Z] - YYYY-MM-DD` and create a new `[Unreleased]`.
- Sections: Added / Changed / Security / Deprecated / Removed / Fixed.
- Cite ADR + doctrine rule where relevant.
- Breaking changes prefixed `BREAKING:` in commit message AND in this entry.
- Sunset notices for deprecations include the Sunset date per RFC 8594.
