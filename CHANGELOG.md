## [Unreleased]

### Added

- **Sprint S6 — GSE Factor Pack.** New `packs/factor/gse/` (manifest + CHARTER) and `backend/packs/factor/gse/` (Go) deliver the second Italian-flagship Factor Pack. Two factor families: (1) the AIB residual mix attribute for Italy (Scope 2 market-based, regulator-recognised baseline for ESRS E1), with year-by-year temporal validity per Rule 90 — 2022/359, 2023/346, 2024/332, 2025/318 g CO₂eq/kWh; (2) Italian renewable-share percentage (Piano 5.0 attestazione input) — 2022/36.0%, 2023/39.5%, 2024/42.1%, 2025/44.5%. Plus per-source renewable shares (hydro, solar, wind, geothermal, biomass) for the 2024 baseline. Static for Phase E; live fetch via `areaclienti.gse.it` deferred to Phase G with circuit breaker per ADR-0009. Refresh cadence: annual Q3 (post AIB August publication) and Q1 (post GSE FER report). Capability `factor-source-gse` declared in `config/required-packs.yaml`. Doctrine refs: Rules 90, 130, 132, 138.
- **Sprint S6 — Italian Region Pack + ISPRA Factor Pack + monthly_consumption Report Pack (vertical slice).** Per Plan §5.4. Three full Packs land together to validate the Pack-extraction pattern end-to-end:
  - `packs/region/it/` (manifest + CHARTER) and `backend/packs/region/it/` (Go) — flagship Region Pack per Rule 88. Profile: timezone Europe/Rome, locale it_IT.UTF-8, currency EUR with comma decimals, default regimes (CSRD wave 2/3, Piano 5.0, Conto Termico, TEE, audit 102/2014, ETS, GDPR, NIS2 Italia, ARERA), authority contact map for GSE / ENEA / ISPRA / Terna / ARERA / Garante / ACN / MIMIT / MASE. Holiday calendar (13 days/year — Italian national + Patrono di Verona Veneto regional); Easter computed via the Meeus/Jones/Butcher Gregorian algorithm; verified against published Easter dates 2008-2038. Regulatory threshold table (16 entries) cited to primary sources per Rule 132 — every value carries a source-document version annotation; Report Packs read via `RegulatoryThreshold(name)` per Rule 139 (thresholds propagated, not duplicated).
  - `packs/factor/ispra/` and `backend/packs/factor/ispra/` — first Italian Factor Pack. Ships static factor table for Phase E: Italian national mix (location-based, Scope 2) for 2022–2026 with year-by-year temporal validity per Rule 90 (268 / 245 / 233 / 225 / 218 g CO₂eq/kWh); Scope 1 stationary combustion (natural gas, LPG, diesel, heavy fuel oil, coal) per D.M. 11/01/2017 Allegato 1; Scope 1 mobile combustion (petrol, diesel road vehicle) per Allegato 2. Refresh signature wired to ISPRA cadence (annual Q2); live-fetch path deferred to Phase G. Tests enforce Rule 132 (every row carries primary source) + Rule 90 (temporal-key uniqueness, non-overlapping intervals).
  - `packs/report/monthly_consumption/` and `backend/packs/report/monthly_consumption/` — reference Report Pack. Pure-function `Build(ctx, period, factors, readings) → Report` per Rule 91 — no `time.Now()`, no env, no `internal/services/` imports. Aggregates by (meter_id, channel_id), looks up Scope 2 factor at period midpoint per Rule 90, emits deterministic JSON with sorted keys per Rule 141. Tests anchor Rule 89 (bit-perfect reproducibility — two consecutive Build calls produce byte-identical Encoded payloads) and verify graceful degradation on missing factor.
  - `engagements/` directory scaffolded at repo root with `README.md` + `.gitkeep`. Per Rule 150, engagement-specific overlays under `engagements/<id>/` are created at Phase 1 (Fork & Bootstrap) of an engagement, not earlier. README documents the layout and the upstream-vs-engagement boundary.
  - `config/required-packs.yaml` populated with the three Italian-flagship capabilities (`region-profile-it`, `holiday-calendar-it`, `regulatory-thresholds-it`, `factor-source-ispra`, `report-builder-monthly_consumption`). Remaining S6/S7/S8 capabilities listed as commented TODOs to track the queue.
  - **ADR-0024 — Pack Go code in `backend/packs/` (interim, single-module phase).** Documents the structural decision: Pack manifest + CHARTER live at repo-root `packs/<kind>/<id>/` per Charter §3.2 discovery; Pack Go code lives at `backend/packs/<kind>/<id>/` to avoid the backend ↔ packs module-cycle problem in the current single-module repo. Schedules supersession to Phase F Sprint S11 with a documented migration procedure (extract contract interfaces to `github.com/greenmetrics/contract` module + relocate Pack code to repo-root).
- **Sprint S5 — SaaS-vocabulary charter-alignment annotations.** Per Plan §5.3.11, added a 2026-04-30 framing note at the top of `docs/SECURITY.md`, `docs/RISK-REGISTER.md`, and `docs/ITALIAN-COMPLIANCE.md` pointing readers at `docs/MODULAR-TEMPLATE-CHARTER.md` §2 (the SaaS → engagement-template pivot) and §11 (single-tenant by default; multi-tenant by configuration as a partner-hosted opt-in). All 14 hook SHAs re-verified via `gh api repos/<owner>/<repo>/commits/<sha>` after the fix. Per Rule 53 (pin every dependency to a digest); Dependabot weekly bumps continue from the corrected baseline.

## How to add an entry

- Add to `[Unreleased]` first.
- On release: rename `[Unreleased]` to `[X.Y.Z] - YYYY-MM-DD` and create a new `[Unreleased]`.
- Sections: Added / Changed / Security / Deprecated / Removed / Fixed.
- Cite ADR + doctrine rule where relevant.
- Breaking changes prefixed `BREAKING:` in commit message AND in this entry.
- Sunset notices for deprecations include the Sunset date per RFC 8594.
